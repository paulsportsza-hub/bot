## Symlink-farm decision (writes redirected to bot-data-shared)

The following bot-tree subtrees become symlinks in the prod runtime
(`/home/paulsportsza/bot-prod/<sub>` → `/home/paulsportsza/bot-data-shared/<sub>`):

| Sub-path | Why writable |
|---|---|
| `data/` | All in-bot SQLite DBs (narrative_cache, logo_cache, edge, mzansiedge, etc.) |
| `reports/` | wave + QA report markdown / JSON |
| `logs/` | rotating log files |
| `bet_log/` | settlement bet log (created on demand by `bet_log_writer`) |
| `__pycache__/` (PYTHONPYCACHEPREFIX) | redirected to `bot-data-shared/pycache/` via systemd Environment so chmod -R u-w on the prod tree does not block bytecode caching |

`scrapers/odds.db` (855 MB) is **not** under bot/, it lives at
`/home/paulsportsza/scrapers/odds.db`. `Path(__file__).parent.parent /

## .venv strategy

`.venv` is symlinked from prod to dev tree (`bot-prod/.venv -> bot/.venv`)
to avoid duplicating ~700 MB per deploy. Trade-off: a `pip install`
mid-deploy in the dev tree would be visible to prod immediately. Operator
discipline: do not install during a deploy window. If a pip-level upgrade
is needed, run it outside business hours and recycle the bot afterwards.
