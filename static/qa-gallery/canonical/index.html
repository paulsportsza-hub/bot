🎨 Design System (universal tokens)

Every canonical card MUST honour these tokens. The unified header pattern (logo + tier-glow halo + brand bar + sparkle layer) is mandatory on edge-tier surfaces. Locked rules carry precedence; deviations require Paul's explicit approval per card.

1 · Header (every edge-tier surface)

  • Logo: mzansiedge-wordmark-dark-transparent.png at height: 28px (detail/breakdown) or 32px (list/index). Loaded via card_data.logo_b64() as base64 data URI. Text fallback only if image absent.
  • Glow zone — covers header + matchup + meta-bar as one unified block (templates: edge_detail, ai_breakdown, tier_lock_upsell):
    • Wrap .header + .matchup (or .match-meta kicker+title for ai_breakdown) + .meta-bar in <div class="upper-glow-zone">. Wrapper has position: relative; isolation: isolate; overflow: hidden.
    • Two glow layers inside the wrapper at z-index: 0:
      • Layer A (.upper-glow, blurred 2px): radial-gradient(ellipse 35% 130% at 92% 50%, {tc}10 0%, {tc}07 50%, transparent 90%)
      • Layer B (.upper-glow-screen, mix-blend-mode: screen): radial-gradient(ellipse 22% 100% at 92% 50%, {tc}1A 0%, {tc}0D 40%, transparent 80%)
    • All inner content (header / matchup / meta-bar) lifted to z-index: 2.
    • Drop the hard dividing line above the meta-bar. Remove border-top: 1px solid #2A2A2A from .meta-bar. Keep border-bottom (separates the unified upper block from the analysis section below — THE PICK / prose / locked-preview).
    • For ai_breakdown specifically: the prod template originally nested .meta-bar inside .match-meta. Canonical pulls .meta-bar out as a sibling of .match-meta (with explicit padding: 8px 18px to replace the lost margin: 0 -18px) so the glow can flow seamlessly to its bottom.
    • Header padding stays at prod values (8px 16px) — do NOT enlarge (eats verdict space on edge_detail).
    • Banned: border-top on .meta-bar (creates the hard dividing line); glow radials extending past the meta-bar's bottom border; header background gradients.
  • Logo-centered glow (for tall-header LIST cards where the wordmark logo is the centerpiece — edge_picks, edge_picks_index_v1, edge_summary): glow radiates from BEHIND the logo (centered top), fading down through the orange-bar.
    • Layer A (.header-glow / .logo-glow, blurred 3px): radial-gradient(ellipse 50% 90% at 50% 25%, #F7931A38 0%, #F7931A18 45%, #F7931A06 70%, transparent 100%)
    • Layer B (.header-glow-screen / .logo-glow-screen, screen-blend): radial-gradient(ellipse 32% 60% at 50% 28%, #F7931A66 0%, #F7931A22 50%, transparent 80%)
    • Container inset: -10px 0 0 0 on Layer A so the glow bleeds slightly above the logo into the card edge.
    • Always brand orange #F7931A (these are multi-tier or non-tier surfaces — orange is the brand stamp).
  • No card border. The card frame is borderless; the glow alone signals the tier in the header row.
  • Brand bar: 2px tall, linear-gradient(90deg, #FF6B00, #F7931A, #FFBE45), margin 0 16px 8px, opacity 0.9. Mandatory below header on every card.
  • Tier badge: pill on header right with tier emoji + label. Background {tier_color}18, border {tier_color}55, box-shadow 0 0 12px {tier_color}30, label colour {tier_color}.

3 · Bookmaker logos (use real PNGs, never plain text)

  • Where logos go vs don't:

7 · Universal laws (apply to every card)

  • No footers. Card terminates with a small black breathing strip below the last content block. No mzansiedge.co.za · Bet responsibly · 18+ footer on any surface.
  • No negative space. Card edge sits 8-12px below last content block. Verify with pixel measurement (trailing_blank ≤ 14px logical).
  • Real bookmaker logos (from /home/paulsportsza/assets/bookmakers/logo-*.png) wherever a bookmaker is mentioned. Logo height 18-24px. Banned: text-only bookie names.
  • Unified header pattern: mzansiedge-wordmark-dark-transparent.png at 28px on every card. Header glow per the glow-type rule above. No header background, no border-bottom. Header padding stays at 8px 16px.
  • Adaptive height (LIST variant) is the default for everything except edge_detail.html + match_detail.html (DETAIL 480×620). LIST cards auto-fit content height via JS measurement.

8 · Typography + colour scaffold

  • Sans: Inter 400/600/700/800; mono: JetBrains Mono 700 (numbers, odds, EV).
  • Body bg #000000; card surface #0A0A0A; section surface #141414; secondary text #B3B3B3 (WCAG AA on dark).
  • Form box colours: W #00E676, D #555555, L #FF5252.
  • Confidence/H2H bars use {tier_color} for fill on accent rows; muted #444 for neutral.

9 · Verdict section — LOCKED to prod (size + position must not change)

  • This is a hard lock. Text limits + verdict-prose generation are calibrated to this exact box. Any size / padding / font / line-height change will break downstream prose-fit assumptions.
  • .verdict-section — flex-grow: 1; min-height: 90px; padding: 10px 14px 12px; background: #141414; border-top: 1px solid #2A2A2A. Identical to bot/card_templates/edge_detail.html:417-424.
  • .verdict-text — font-size: 15px; color: #DDDDDD; line-height: 1.6; margin-top: 5px; white-space: pre-line. Identical to prod.
  • .verdict-section .section-hdr — font-size: 11px.
  • Banned: changing min-height, adding max-height, line-clamp, or tier-coloured background/border on the verdict — those alter the size/position of the box that prose generators are calibrated to.
  • Verification: diff <canonical-verdict-css> <prod-verdict-css> must produce zero output. Tested + locked 2026-04-25 ✓ (Paul approved Diamond 90 / Gold 84 / Silver 108 / Bronze 109 vs prod baseline 88 — within ±10px tolerance).
  • CRITICAL: verdict-section uses flex-grow: 1 — it fills remaining space inside the 620px frame. Therefore any padding or sizing change in the sections above (header, matchup, pick, chips, signals, H2H) will shrink the verdict's effective rendered height, even though its CSS is unchanged. Pixel measurement check (Diamond/Gold should both land near 88px logical): python3 -c "from PIL import Image; import numpy as np; img=np.array(Image.open('canonical/edge_detail_gold.png').convert('RGB')); h,w,_=img.shape; col=img[:,w//2]; m=np.all(np.abs(col.astype(int)-np.array([20,20,20]))<=5,axis=1); y=h-1; \nwhile y>=0 and m[y]: y-=1\nprint('verdict_height_logical=', (h-(y+1))//2)" Tolerance: ±10px logical vs prod's 88px. Banned: increasing header padding, adding brand-bar between header and matchup, increasing chip/signals padding — all eat verdict space.

10 · Footer — REMOVED (universal law §7)

  • No footer on any card. The previous mzansiedge.co.za · Bet responsibly · 18+ footer is dropped from every surface. Card terminates with the black breathing strip rule (§7).
  • Responsible-gambling messaging surfaces only on monthly reports + trial messages + re-engagement nudges (per CLAUDE.md §Notification Content Laws), not on UI cards.
